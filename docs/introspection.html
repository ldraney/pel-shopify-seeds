How It Works

Step 1: GraphQL Introspection Query

The script uses GraphQL's built-in introspection system to query type metadata:

query IntrospectType($typeName: String!) {
  __type(name: $typeName) {
    name
    kind
    description
    fields {
      name
      description
      type {
        name
        kind
        ofType { name kind ofType { name kind } }
      }
    }
  }
}

This query returns the complete structure of any type in the Shopify schema.

Step 2: Types Introspected

The script queries these product-related types:

TypeQueryAPI ScopePurpose
Product products / product(id:) read_products Product details, title, description, status
ProductVariant productVariants read_products SKU, price, inventory, options
Collection collections read_products Manual and smart collections
InventoryItem inventoryItems read_inventory HS codes, country of origin, tracking
InventoryLevel via inventoryItem.inventoryLevels read_inventory Quantity per location
Location locations read_locations Warehouses, stores, addresses
Metafield via parent object varies Custom fields on any resource

Step 3: Generate types.js

For each type, the script extracts:

  • Field name - The GraphQL field name
  • Type - Full type signature (e.g., String!, [ProductVariant!]!)
  • Base type - Underlying scalar or object type
  • Required - Whether field is non-null
  • List - Whether field returns an array
  • Description - Documentation from Shopify

Example output for a field:

{
  "handle": {
    "type": "String!",
    "baseType": "String",
    "required": true,
    "list": false,
    "description": "A unique human-friendly string of the product's title."
  }
}

Usage

Running the Introspection Script

# Set environment variables
export SHOPIFY_STORE_URL=your-store.myshopify.com
export SHOPIFY_ACCESS_TOKEN=shpat_xxxxx

# Run introspection
node src/introspect.js

Or inline:

SHOPIFY_STORE_URL=your-store.myshopify.com \
SHOPIFY_ACCESS_TOKEN=shpat_xxx \
node src/introspect.js
Note: The access token needs at minimum these scopes: read_products, read_inventory, read_locations

Output

The script generates api/types.js with this structure:

module.exports = {
  generatedAt: "2025-12-07T...",

Table Creation from types.js

Views reads the field definitions and creates matching SQLite tables:

// Example: Creating collections table from types.js
const { types } = require('./types.js');
const collectionFields = types.Collection.fields;

// Generate CREATE TABLE statement
let sql = 'CREATE TABLE collections (\n';
for (const [fieldName, fieldDef] of Object.entries(collectionFields)) {
  const sqlType = mapGraphQLToSQLite(fieldDef.baseType);
  sql += `  ${toSnakeCase(fieldName)} ${sqlType},\n`;
}
sql += ')';

// Result:
// CREATE TABLE collections (
//   id TEXT PRIMARY KEY,
//   handle TEXT,
//   title TEXT,
//   description_html TEXT,
//   ...
// );

Always accurate Schema comes directly from Shopify's live API
Complete All fields captured, not just manually documented ones
Self-documenting Descriptions included from API documentation

When to regenerate:
  • Upgrading Shopify API version
  • Shopify adds new fields you need
  • Building for a different store with custom types
